Tell us the use cases of null and undefined

Ans: null is a value that represents intentional absence of any object value. It's often used to indicate that a variable intentionally does not have a value. And undefined is a value that represents an uninitialized or unassigned variable, as well as a missing property in an object or an undefined function parameter.

What do you mean by REST API?

Ans: A REST API is an API that conforms to the design principles of the REST, or representational state transfer architectural style.
